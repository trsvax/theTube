# Events Endpoint — Spec

## What this is

An `/events/` path on CloudFront that accepts any GET request and returns 202. CloudFront logs the request. The URL is the log entry. Write immediately, process later.

## What triggers it

Any HTTP GET to `https://thetube.today/events/...`

## What it reads

Nothing. It receives the request and returns immediately.

## What it produces

- A 202 Accepted response (no body)
- A line in the CloudFront access log containing the full URI (path + query string)

## Where it lives

The handler is added to the existing CloudFront Function (`cf-short-urls.js`), generated by `scripts/build-cf-function.mjs`. No new function, no new behavior, no new infrastructure.

---

## Requirements

### Requirement 1: CloudFront Function handles /events/

WHEN a request URI starts with `/events/`, THE CloudFront Function SHALL return a 202 Accepted response with no body, `cache-control: no-store`, and `access-control-allow-origin: https://thetube.today`.

### Requirement 2: Events are logged

WHEN a request to `/events/...` is processed, CloudFront SHALL log the full URI (path + query string) in its standard access logs. No additional configuration needed — CloudFront logs all requests by default when access logging is enabled.

### Requirement 3: Access logging is enabled

THE CloudFront distribution SHALL have standard access logging enabled, writing to an S3 bucket. If not already enabled, enable it.

### Requirement 4: No authentication required

THE `/events/` path SHALL accept requests from any visitor — authenticated or not. If the visitor is logged in, the JWT cookie is included in the request automatically and logged. No gating.

### Requirement 5: URL format

Events use path hierarchy for the event type and query string for the content:

```
/events/{category}/{sub}?key=value&key=value
```

Examples:
```
/events/link/dead?url=https://example.com/gone&post=my-post
/events/deploy/done?commit=abc1234&duration=28s
/events/error/js?msg=TypeError&post=my-post
```

### Requirement 6: Build script generates the handler

THE `scripts/build-cf-function.mjs` SHALL include the `/events/` handler in the generated `cf-short-urls.js`. The handler checks `uri.startsWith("/events/")` before any other routing logic.
