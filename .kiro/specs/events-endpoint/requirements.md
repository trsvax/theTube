# Write Endpoint (`/tube/`) — Spec

## What this is

A `/tube/` path on CloudFront that accepts POST requests and returns 202. CloudFront logs the request. The URL is the log entry. Write immediately, process later. If the request includes a body, Lambda handles it instead.

## What triggers it

Any HTTP POST to `https://thetube.today/tube/...`

## What it reads

Nothing (default path). The CloudFront Function receives the request and returns immediately. If a body is present, Lambda reads it.

## What it produces

- Default (no body): A 202 Noted response (no body) + a line in the CloudFront access log containing the full URI
- With body: Lambda response (200/201/202 with optional body)

## Where it lives

The handler is in the CloudFront Function (`cf-short-urls.js`), generated by `scripts/build-cf-function.mjs`. No new function, no new infrastructure.

---

## Requirements

### Requirement 1: CloudFront Function handles /tube/ with query string

WHEN a POST request URI starts with `/tube/` and has a query string (`?` present), THE CloudFront Function SHALL return a 202 Noted response with no body, `cache-control: no-store`, and `access-control-allow-origin: https://thetube.today`.

### Requirement 2: Requests without query string pass to Lambda

WHEN a POST request URI starts with `/tube/` and has no query string (no `?`), THE request SHALL be forwarded to Lambda@Edge for processing. The Lambda validates auth and processes the request body.

### Requirement 3: Writes are logged

WHEN a request to `/w/...` is processed, CloudFront SHALL log the full URI (path + query string) in its standard access logs. This applies to both paths.

### Requirement 4: Access logging is enabled

THE CloudFront distribution SHALL have standard access logging enabled, writing to an S3 bucket.

### Requirement 5: No authentication required for batch path

THE `/tube/` path with a query string SHALL accept requests from any visitor — authenticated or not. If the visitor is logged in, the JWT cookie is included in the request automatically and logged. No gating on the batch path.

### Requirement 6: Authentication required for Lambda path

WHEN a request to `/tube/` has no query string (forwarded to Lambda), Lambda@Edge SHALL validate the JWT before processing. No valid JWT → 401. Insufficient role → 403.

### Requirement 7: URL format

Writes use path hierarchy for namespace/verb and query string for data:

```
/tube/{namespace}/{verb}?key=value&key=value
```

Examples (batch path — `?` present):

```
/tube/comment/add?post=my-post&body=great+post&author=barry
/tube/contact/submit?name=barry&email=x@y.com
/tube/reaction/add?post=my-post&type=like
/tube/deploy/done?
/tube/error/js?msg=TypeError&post=my-post
```

Examples (Lambda path — no `?`):

```
/tube/comment/add
/tube/booking/create
```

### Requirement 8: POST only

THE `/w/` path SHALL only accept POST requests. GET requests to `/w/` SHALL return 405 Method Not Allowed. POST prevents CDN caching — every request hits the function.

### Requirement 9: The `?` is the routing signal

THE CloudFront Function SHALL check for the presence of a query string. If present (even empty, e.g. `/tube/deploy/done?`), return 202 immediately. If absent, pass through to Lambda@Edge.

### Requirement 10: Build script generates the handler

THE `scripts/build-cf-function.mjs` SHALL include the `/tube/` handler in the generated `cf-short-urls.js`. The handler checks `uri.startsWith("/tube/")` before any other routing logic.

### Requirement 11: Failure mode for Lambda path

IF Lambda returns a 5xx error for a request without a query string, the response SHALL be `503 Service Unavailable` with a `Retry-After` header. The client retries.

## What's NOT in scope

- Processing the logs (future — Lambda reads logs, produces artifacts)
- Querying with Athena (future — when you need to ask questions)
- Plugin-specific Lambda implementations (each plugin provides its own)

## Migration

This replaces the previous `/events/` spec. The `/events/` path is deprecated. Existing references should migrate to `/tube/`.

- `/events/{category}/{sub}?data` → `POST /tube/{category}/{sub}?data`
- `/fastevent/{operation}` with body → `POST /tube/{namespace}/{verb}` with body
