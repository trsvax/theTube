# The Platform Filesystem

Everything is a file at a URL. The platform is a POSIX-like filesystem on a CDN.

---

## Syscall mapping

| POSIX | Platform | Implementation |
|---|---|---|
| `open()` | `/w/{ns}/open?...` | Get a token (file descriptor). Checks permissions, returns a scoped handle. |
| `write()` | `POST /w/{ns}/{verb}?...` | Write data. Token required. `?` = batch (202). No `?` = sync (Lambda). |
| `create()` | Processor writes `/{ns}/{id}.json` | New file. Immutable once written. `If-None-Match: *`. |
| `read()` | `GET /{path}` | Fetch a file. CloudFront serves from cache or origin. |
| `readdir()` / `ls` | `GET /{path}/index.json` | List a directory. Static file or Lambda-generated. |
| `stat()` | `HEAD /{path}` | Metadata: Last-Modified, Content-Length, ETag. |
| `seek()` + `read()` | `GET /{path}` + `Range: bytes=N-M` | Random access. S3 supports byte-range requests. |
| `close()` | Token expires | Implicit. No explicit close needed. |
| `unlink()` | `/w/{ns}/delete?...` | Soft delete: writes `{"deleted": true}`. Log has history. |
| `rename()` | Not needed | Files are addressed by ID, not name. Content can change slugs without moving files. |
| `mkdir()` | Implicit | Directories exist when files exist in them. No explicit creation. |
| `chmod()` | IAM + CloudFront behaviors | Change who can access a path. Platform-level, not per-file. |
| `fork()` | New repo | Isolated process. Own namespace, own Lambda, own IAM, own S3 prefix. |
| `append()` | Create a new file in the directory | `path/1`, `path/2`, `path/3`. The directory is the append log. |
| `pipe()` | Logs → processor → files | Data flows: edge logs events, processor reads logs, writes files. |
| `link()` | A file containing a URL | Not built yet. A file could redirect to another file. |

---

## Principles

**Files are immutable.** Written once, never modified. Edits are new files (versions). Deletes are soft (marker file). The log has the full history.

**Directories grow by adding files.** "Append" is creating a new file in the directory, not modifying an existing file. No read-modify-write on the hot path.

**`index.json` is `ls`.** Every directory has one. It lists the contents. It's either a static file (generated by a processor) or a Lambda response (computed on read). Same URL either way.

**The CDN is the kernel.** CloudFront routes URLs to implementations (S3, Lambda, cache). The client doesn't know what's behind a URL. The kernel decides.

**Compute is pluggable.** Any URL can be backed by a static file today and a Lambda tomorrow. The interface (URL + response format) never changes. Static is the default. Lambda is the upgrade.

**Namespaces are isolation boundaries.** Each namespace (`/comments/`, `/contact/`, `/admin/`) is a separate process. Own repo, own Lambda, own IAM, own S3 prefix. Can't touch other namespaces.

**Tokens are file descriptors.** `open()` returns a scoped, signed token. The token carries: which path, what permissions, how many uses, when it expires. The processor validates the token, not the client's intent.

**Logs are the journal.** Every write is logged (CloudFront access logs). The log is immutable, append-only, and the source of truth. Files on S3 are a projection — rebuildable from the log.

## The read path

```
Client → GET /path/index.json → CloudFront
  → Cache hit? Return cached.
  → Cache miss? → Origin (S3 file or Lambda)
    → S3: return static index.json
    → Lambda: read index.json + list new files + merge + return
  → Cache the response. Next reader gets it free.
```

Graceful degradation: if Lambda is down, CloudFront serves stale cache. Always works. Just maybe not fresh.

---

## The write path

```
Client → POST /w/{ns}/{verb}?data... → CloudFront Function
  → Shape valid? (token present, size ok, params ok)
    → No: 4xx. Not logged.
    → Yes: 202. Logged.
  → Processor (Lambda, triggered by log delivery)
    → Validate token, check permissions, check content
    → Write file to /{ns}/{path}/{id}.json
    → (Optional) Trigger index rebuild
```

The edge is fast and dumb. The processor is slow and smart. The log connects them.

## Trust model

| Level | Issued by | Proves | Behavior |
|---|---|---|---|
| None | — | Nothing | Rejected at edge (no token = 403) |
| Low | Build-time HMAC in HTML | Client loaded the page | Moderated |
| Medium | Rotated token file | Client loaded the page recently | Auto-filtered |
| High | `open()` Lambda | Client asked permission now | Written immediately |
| Admin | Auth header (no `?`) | Verified identity | Sync response, full access |

The `?` is the security boundary: present = data in URL = logged (low/medium trust). Absent = data in body = not logged (high/admin trust, credentials stay off disk).

---

## What this replaces

- Databases → files + indexes
- Servers → Lambda (stateless, ephemeral)
- Message queues → logs + S3 events
- Auth middleware → CDN layer (CloudFront Functions + Lambda@Edge)
- Caching layers → CDN (built in)
- Load balancers → CDN (built in)
- Monitoring → logs (already there)
